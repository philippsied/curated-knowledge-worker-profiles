# UI/UX Plugin

Eine kuratierte Sammlung hochwertiger Skills fuer UI- und UX-Aufgaben – von der Gestaltung von Design-Systemen bis zur Typografie, Forschung und Barrierefreiheit.

## Enthaltene Skills

| Skill | Zweck |
|-------|-------|
| **bencium-controlled-ux-designer** | Kollaboratives UI/UX-Design – fragt vor jeder Designentscheidung nach. Fuer Projekte mit klar definiertem Brand. |
| **bencium-innovative-ux-designer** | Kreatives UI/UX-Design – setzt mutig auf eine unverwechselbare Richtung. Fuer offene, kreative Aufgaben. |
| **bencium-impact-designer** | Produktionsreifes Frontend-Design mit erweitertem Aesthetik-Katalog (40+ Stilrichtungen). |
| **bencium-code-conventions** | Code-Stil, Tech-Stack und Workflow-Konventionen fuer React/Next.js-Projekte. |
| **bencium-aeo** | Answer Engine Optimization – Inhalte fuer KI-Zitierungen (ChatGPT, Claude, Gemini) optimieren. |
| **ui-ux-pro-max** | UI/UX-Design-Intelligenz: 50+ Stile, 161 Farbpaletten, 57 Schriftpaare, 99 UX-Richtlinien, 25 Diagrammtypen uebergreifend fuer 10 Stacks (React, Next.js, Vue, SwiftUI, Flutter u. a.). |
| **ckm-design** | Umfassendes Design-Skill: Brand Identity, Logos (55 Stile, Gemini AI), Corporate Identity (50 Deliverables), HTML-Praesentationen, Banner, Icons, Social-Media-Bilder. |
| **ckm-design-system** | Token-Architektur (Primitive → Semantik → Komponente), CSS-Variablen, Spacing-/Typografie-Skalen, Komponentenspezifikationen. |
| **ckm-ui-styling** | Schoene, barrierefreie UIs mit shadcn/ui, Tailwind CSS und Canvas-basiertem Visual Design. Dark Mode, responsive Layouts, Themes. |
| **ckm-banner-design** | Banner-Design fuer Social Media, Ads, Website-Heroes und Print – mehrere Art-Direction-Optionen mit KI-generierten Visuals. |
| **ckm-brand** | Brand Voice, visuelle Identitaet, Messaging-Frameworks, Asset-Management und Brand-Consistency. |
| **ckm-slides** | Strategische HTML-Praesentationen mit Chart.js, Design Tokens, Responsive Layouts und Copywriting-Formeln. |
| **design-system** | Design-System auditieren, dokumentieren oder erweitern: Naming, Tokens, Komponentenvarianten, Zustaende, Accessibility-Notizen. |
| **design-audit** | Systematischer visueller Audit bestehender Apps in drei Phasen (Kritisch → Verfeinerung → Polish). |
| **design-critique** | Strukturiertes Designfeedback zu Usability, Hierarchie und Konsistenz – fuer jede Projektphase von Exploration bis Final. |
| **design-handoff** | Entwickler-Handoff-Spezifikationen: Layout, Design Tokens, Komponentenprops, Interaktionszustaende, Breakpoints, Animationsdetails. |
| **accessibility-review** | WCAG-2.1-AA-Audit: Farbkontrast, Tastaturnavigation, Touch-Target-Groessen, Screen-Reader-Verhalten. |
| **web-design-guidelines** | UI-Code auf Einhaltung der Web Interface Guidelines pruefen (Accessibility, UX Best Practices). |
| **typography** | Professionelle Typografie-Regeln fuer UI – basierend auf Matthew Butterick's *Practical Typography*. |
| **ux-copy** | UX-Texte schreiben oder reviewen: Microcopy, Fehlermeldungen, Empty States, CTAs, Onboarding-Texte. |
| **user-research** | User Research planen, durchfuehren und auswerten: Interview-Leitfaeden, Usability-Tests, Survey-Design. |
| **research-synthesis** | User-Research-Daten (Interviews, Surveys, Usability-Tests) zu Themes, Insights und priorisierten Handlungsempfehlungen verdichten. |
| **adaptive-communication** | Kommunikationsstil automatisch an den Kontext anpassen (transaktional vs. relational). |
| **human-architect-mindset** | Architekturdenken: Domain Modeling, Systems Thinking, Constraint Navigation, KI-Decomposition. |

## Voraussetzungen

- Einige Skills (z. B. `human-architect-mindset`) verweisen auf den **superpowers**-Plugin fuer erweitertes Brainstorming und Plan-Erstellung.
- `ckm-banner-design`, `ckm-design` und `ckm-ui-styling` setzen `ui-ux-pro-max` voraus.
