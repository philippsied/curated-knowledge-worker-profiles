# Design Plugin

Eine kuratierte Sammlung hochwertiger Skills fuer UI- und UX-Aufgaben – von der Gestaltung von Design-Systemen bis zur Typografie, Forschung und Barrierefreiheit.

## Enthaltene Skills

| Skill | Beschreibung | Herkunft |
|-------|-------------|----------|
| **bencium-controlled-ux-designer** | Experten-UI/UX-Designbegleitung fuer einzigartige, barrierefreie Interfaces. Fragt vor jeder Designentscheidung nach. Fuer Projekte mit klar definiertem Brand. | Bence (bencium) |
| **bencium-innovative-ux-designer** | Unverwechselbare, produktionsreife Frontend-Interfaces mit hoher Designqualitaet. Setzt mutig auf eine klar definierte Richtung. Vermeidet generische KI-Aesthetik. | Bence (bencium) |
| **bencium-impact-designer** | Unverwechselbare, produktionsreife Frontend-Interfaces mit erweitertem Aesthetik-Katalog (40+ Stilrichtungen). Vermeidet generische KI-Aesthetik. | Bence (bencium), basiert auf Anthropics Frontend Designer Skill |
| **bencium-code-conventions** | Code-Stil, Tech-Stack und Workflow-Konventionen fuer React/Next.js-Projekte. | Bence (bencium) |
| **bencium-aeo** | AEO-optimierte Inhalte fuer KI-Sichtbarkeit (ChatGPT, Claude, Gemini, AI Overviews). FAQ-Schemas, Evidence Panels und Analyse der LLM-Extraktionsbereitschaft von Inhalten. | Bence (bencium) |
| **ui-ux-pro-max** | UI/UX-Design-Intelligenz: 50+ Stile, 161 Farbpaletten, 57 Schriftpaare, 99 UX-Richtlinien, 25 Diagrammtypen fuer 10 Stacks (React, Next.js, Vue, Svelte, SwiftUI, React Native, Flutter, Tailwind, shadcn/ui, HTML/CSS). | – |
| **ckm-design** | Umfassendes Design-Skill: Brand Identity, Design Tokens, UI-Styling, Logos (55 Stile, Gemini AI), Corporate Identity (50 Deliverables, CIP-Mockups), HTML-Praesentationen (Chart.js), Banner (22 Stile), Icons (15 Stile, SVG, Gemini), Social-Media-Bilder. | claudekit |
| **ckm-design-system** | Token-Architektur (Primitive → Semantik → Komponente), CSS-Variablen, Spacing-/Typografie-Skalen, Komponentenspezifikationen und strategische Slide-Erstellung. | claudekit |
| **ckm-ui-styling** | Schoene, barrierefreie UIs mit shadcn/ui (Radix UI + Tailwind), Tailwind CSS Utility-First-Styling und Canvas-basiertem Visual Design. Dark Mode, responsive Layouts, Themes. | claudekit |
| **ckm-banner-design** | Banner fuer Social Media, Ads, Website-Heroes, kreative Assets und Print – mehrere Art-Direction-Optionen mit KI-generierten Visuals. | claudekit |
| **ckm-brand** | Brand Voice, visuelle Identitaet, Messaging-Frameworks, Asset-Management und Brand-Consistency. | claudekit |
| **ckm-slides** | Strategische HTML-Praesentationen mit Chart.js, Design Tokens, Responsive Layouts, Copywriting-Formeln und kontextuellen Slide-Strategien. | claudekit |
| **design-system** | Design-System auditieren, dokumentieren oder erweitern: Naming-Konsistenz, hardcodierte Werte, Komponentenvarianten, Zustaende, Accessibility-Notizen, neue Pattern. | – |
| **design-audit** | Systematischer visueller Audit bestehender Apps – erzeugt phasenweise, umsetzungsreife Designplaene (Kritisch → Verfeinerung → Polish). | – |
| **design-critique** | Strukturiertes Designfeedback zu Usability, Hierarchie und Konsistenz – fuer jede Projektphase von Exploration bis Final. | – |
| **design-handoff** | Entwickler-Handoff-Spezifikationen: Layout, Design Tokens, Komponentenprops, Interaktionszustaende, responsive Breakpoints, Edge Cases und Animationsdetails. | – |
| **accessibility-review** | WCAG-2.1-AA-Audit fuer ein Design oder eine Seite. | – |
| **web-design-guidelines** | UI-Code auf Einhaltung der Web Interface Guidelines pruefen: Accessibility, UX Best Practices, Design-Audit, Site Review. | Vercel (vercel-labs) |
| **typography** | Professionelle Typografie-Regeln fuer UI-Design, Web-Apps, Software-Interfaces und alle bildschirmbasierten Texte – destilliert aus Matthew Butterick's *Practical Typography*. | Matthew Butterick |
| **ux-copy** | UX-Texte schreiben oder reviewen: Microcopy, Fehlermeldungen, Empty States, CTAs, Onboarding-Texte. | – |
| **user-research** | User Research planen, durchfuehren und auswerten: Interview-Leitfaeden, Usability-Tests, Survey-Design. | – |
| **research-synthesis** | User-Research-Daten (Interviews, Surveys, Usability-Tests) zu Themes, Insights und priorisierten Handlungsempfehlungen verdichten. | – |
| **adaptive-communication** | Kommunikationsstil automatisch an den Kontext anpassen – bei unklarem Intent, offenem Framing oder wenn Erkundung statt direkter Antwort gefragt ist. | – |
| **human-architect-mindset** | Systematisches Architekturdenken fuer menschliche Kernkompetenzen: Domain Modeling, Systems Thinking, Constraint Navigation, KI-Decomposition. | – |

## Voraussetzungen

- Einige Skills (z. B. `human-architect-mindset`) verweisen auf den **superpowers**-Plugin fuer erweitertes Brainstorming und Plan-Erstellung.
- `ckm-banner-design`, `ckm-design` und `ckm-ui-styling` setzen `ui-ux-pro-max` voraus.
